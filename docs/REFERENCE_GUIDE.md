# 参考站点：FPGA/ASIC 方向的个人技术展示（可学习清单）

这份文档收集一些在 FPGA / 数字设计 / ASIC 验证圈子里较有代表性的个人站点或技术内容站点，并总结“值得学习的点”，用于你迭代自己的作品集网站。

> 说明：
>
> - 这些链接用于学习表达方式与信息组织方式。
> - 是否开源、是否允许转载，以对方站点声明为准。

---

## 1) 个人技术展示 / 博客（强烈推荐参考）

### ZipCPU（Dan Gisselquist）

- 链接：[https://zipcpu.com/](https://zipcpu.com/)
- 值得学习的点：
  - 文章主题长期聚焦在数字设计真实问题（AXI、DDR、调试、形式验证等），形成“专家标签”。
  - 写作结构稳定：问题 → 背景 → 方案 → 验证/复现 → 结论/经验。
  - 很强调“如何验证它真的工作”，天然带来可信度。
  - 适合你对标的内容形式：
    - 每个重点项目页开头用 3 行摘要说明“做了什么/为什么重要/结果是什么”。
    - 用 1–2 个关键图 + 1 个关键指标 + 1 个验证方法作为最小可信单元。

### Clifford Wolf（开源 EDA / 工具作者）

- 链接：[https://www.clifford.at/](https://www.clifford.at/)
- 值得学习的点：
  - 主页像“作品目录（Index）”，以分类导航为主，让人快速找到项目入口。
  - 适合你对标的内容形式：
    - 首页做一个“Projects Index（按主题分类）”，每项只放 2–3 个最关键入口：项目介绍页 / Demo（图或视频）/ 公开材料（报告或博客）。

### Nandland（Russell Merrick）

- 链接：[https://nandland.com/](https://nandland.com/)
- 值得学习的点：
  - 把内容组织成学习路径（教程/视频/导航），用户能“从零走到能做东西”。
  - 适合你对标的内容形式：
    - 把博客做成“精选 8 篇 + 主题路径”，而不是把所有文章平铺在首页。
    - 每篇文章提供“你会得到什么”和“典型踩坑点”，对面试也有帮助。

## 5) 你可以如何把这些“学到的点”落到自己的网站

### 5.1 首页（10 秒建立信任）

- 建议结构：
  - 一句话定位（岗位 + 方向 + 亮点）
  - 2–3 个重点项目卡片（每个卡片含 1 个指标）
  - 精选博客入口（6–10 篇，不要堆满）

### 5.2 重点项目页（最小可信单元）

每个重点项目页尽量包含以下“最小可信单元”（信息不必多，但要完整）：

- 3 行摘要：目标 / 你的贡献 / 结果
- 1 张架构图：模块边界清晰
- 2 个指标：频率/吞吐/资源/时延/覆盖率等（可公开）
- 1 个验证证据：仿真/ILA/回归/对照实验（脱敏）

### 5.3 博客精选（从“数量”转为“命中率”）

- 首页精选建议控制 8 篇左右：
  - AXI、DDR/MIG、时序约束、复位、高速接口、流水线、代码风格等
- 每篇文章开头提供：
  - 你会得到什么（1–2 句）
  - 适用场景
  - 常见坑点（至少 1 条）
